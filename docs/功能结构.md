# 随芯调功能结构

这份结构按用户任务组织，而不是按底层模块组织。后续 UI 和功能入口优先按这棵树推进。

说明：中文界面中优先使用“应用配置”，必要时在文档或开发说明中补充 `Profile`，用于对应代码和 JSON 文件里的命名。

```text
随芯调
├─ 控制台
│  ├─ 当前前台应用
│  ├─ 立即开启 / 关闭自动调度
│  ├─ 一键回滚恢复
│  ├─ 新手引导
│  │  ├─ 1. 导入或创建应用配置
│  │  ├─ 2. 开启自动调度
│  │  └─ 3. 异常时一键回滚
│  ├─ 当前调度状态
│  │  ├─ 匹配到的应用配置
│  │  ├─ 当前电源方案
│  │  ├─ 当前优先级
│  │  └─ 基础指标摘要
│  └─ 最近调度结果
│
├─ 调度配置
│  ├─ 社区资源
│  │  ├─ 机型策略包
│  │  └─ 应用配置
│  ├─ 本地库
│  │  ├─ 机型策略包
│  │  └─ 应用配置
│  ├─ 应用配置编辑器
│  │  ├─ 基础信息
│  │  ├─ 软件匹配规则
│  │  ├─ 电源与处理器策略
│  │  ├─ 前台优先级
│  │  ├─ 后台程序策略
│  │  ├─ 显卡扩展策略
│  │  ├─ 校验结果
│  │  └─ 历史版本
│  └─ 导入导出
│     ├─ 应用配置导入导出
│     └─ 迁移包导入导出
│
├─ 诊断中心
│  ├─ 能力检测
│  ├─ 设备指纹
│  ├─ 显卡扩展准备状态
│  ├─ 存储状态
│  └─ 最近日志
│
├─ 美化
│  ├─ 窗口背景
│  ├─ 侧边栏背景
│  ├─ 主内容背景
│  ├─ 主题预设
│  ├─ 界面元素颜色
│  ├─ 卡片透明度
│  └─ 背景强度
│
└─ 设置
   ├─ 调度行为
   │  ├─ 默认启用自动调度
   │  └─ 监控频率
   ├─ 安全保护
   │  ├─ 异常退出后暂停调度
   │  ├─ 异常退出后自动回滚
   │  └─ 多次异常后禁用应用配置
   ├─ 启动行为
   │  ├─ 开机自启
   │  └─ 启动后自动开启自动调度
   └─ 文件位置
      ├─ 程序目录
      ├─ 应用配置目录
      ├─ 策略包目录
      ├─ 运行数据
      ├─ 社区缓存
      ├─ 数据库
      ├─ 语言包
      └─ 日志
```

当前优先级：

```text
1. 先保证控制台能看懂当前状态和下一步操作。
2. 再把调度配置拆清楚：社区资源、本地库、编辑器。
3. 诊断中心只放排障和检测信息，不混入配置入口。
4. 美化只放外观选项，不混入调度设置。
5. 设置只放长期偏好、安全保护和文件位置。
```
