# 安全策略

随芯调会读取和调整 Windows 电源方案、进程优先级等本机状态。任何安全问题、危险默认行为、不可回滚状态污染或隐私泄露风险都应优先处理。

## 重点关注范围

- 电源方案或高级电源设置无法恢复。
- 进程优先级调整影响系统关键进程。
- 异常退出、崩溃或快速切换前台窗口后留下污染状态。
- 导入的 Profile 或社区策略绕过本地安全保护。
- 日志、迁移包、设备指纹或导出文件包含不应公开的个人信息。
- GPU、驱动或厂商扩展能力产生不可解释或不可回滚的系统影响。

## 报告方式

当前项目处于早期阶段。如果你发现安全问题，请优先通过 GitHub Issue 提交最小复现信息，并避免公开敏感数据。

报告建议包含：

- 复现步骤。
- 操作系统版本和供电状态。
- 命中的 Profile 或相关配置片段。
- 预期行为和实际行为。
- 是否可以通过一键回滚恢复。
- 相关日志片段，提交前请删除用户名、设备序列号、私有路径、截图隐私信息。

如果问题包含可被滥用的细节，请先只描述影响范围和联系方式，不要直接贴完整利用步骤。

## 处理原则

- 安全问题优先于性能和新功能。
- 默认行为必须保守。
- 修复应包含测试或可验证的复现说明。
- 修复涉及调度、回滚或导入策略时，需要同步更新文档。
